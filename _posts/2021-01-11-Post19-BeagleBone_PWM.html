Introduction

In this post, I start with the PWM technique applied to control the brightness of a LED. Rembering the PWM is a technique that lets us emulate an analog signal in a digital pin. This is done by changing the time that a signal is On in a fixed period of time. The ratio between the time ON and OFF is named duty cycle. The more duty cycle, the more is the "analog" average voltage measured on the digital pin.

Circuit and components

The circuit can be seen in Figure 1. Please keep in mind that the BeagleBone works at 3.3V and not 5V like microcontrollers as Arduino. It is so muy important to avoid damage to the board, especially when you are working with buttons or digital inputs in general.

The components are:
  • 5 Resistor of 1KΩ
  • 1 LED
  • Jumpers male-male to make the connections

Coding

The methods coded to the C++ class PWM are:

  • Private methods:
    • WriteFile(string, string, int)
    • Enable()
    • Disable()
  • Public methods:
    • GetPeriod()
    • GetDutyCycle()
    • SetPeriod(int)
    • SetDutyCycle(int)
    • Delayms(int)

The WriteFile(string, string, int) method access to the file system path that control the pwm behavior on the pin at "/sys/devices/platform/ocp/".

The Enable() method activates the PWM on the pin setting the enable property to 1. This method is called in the Constructor.

The Disable() method deactivates the PWM on the pin setting the enable property to 0. This method is called in the Destructor.

The GetPeriod() method, returns the last set pin's period.

The GetDutyCycle() method, returns the last set pin's duty cycle.

The SetPeriod(int) method, receives an int number and set the period property of the pin in the sys file system.

The SetDutyCycle(int) method, receives an integer number between 0 and 100 and converts it as a percentage of the period before to set the duty_cycle property of the pin in the sys file system. The method checks if the number is less or greater than that range, and set the duty_cycle property as 0 or 100, respectively.

Code:

First an PWM class objet is declared, for example:

{% highlight cpp%} PWM pwmBlueLedPin(P8_13); {% endhighlight %}

After, a variable is declared, initialized and used to set the initial pwm value.

{% highlight cpp%} int pwmValue = 50; pwmBlueLedPin.SetDutyCycle(pwmValue); {% endhighlight %}

Finally, this pwm value can be updated in this manner:

{% highlight cpp%} pwmBlueLedPin.SetDutyCycle(pwmValue += 10); {% endhighlight %}
