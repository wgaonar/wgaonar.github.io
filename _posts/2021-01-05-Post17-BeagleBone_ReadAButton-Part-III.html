Introduction

In this post, I will continue to show you how to read a button with the BeagleBone Black. In the second entry I showed how to wait for the press on a button stoping the main program until a button has been pressed.

On the other hand, in this post, I show a new method named WhenButtonWasPressed(*) that executes a USER customized function when the button has been pressed.

Circuit and components

The circuit can be seen in Figure 1. Please keep in mind that the BeagleBone works at 3.3V and not 5V like microcontrollers as Arduino. It is so muy important to avoid damage to the board, especially when you are working with buttons or digital inputs in general.

The components are:
  • 1 Resistor of 10KΩ as a pull-down resistor
  • 1 Push button of 12mm
  • Jumpers male-male to make the connections

Coding

This method was added to the C++ class BUTTON which is derived from the GPIOclass.

  • Public methods:
    • WhenButtonWasPressed(*)

In the BUTTON class has to be defined a callbackType for the function pointer that will be used as an argument to the method. This can be done in the header file BUTTON.h with the next line:

{% highlight cpp%} using callbackType = int (*)(); {% endhighlight %}

The WhenButtonWasPressed(*) method, first checks receives a user customized function when the button has been pressed and creates a thread with a function pointer to that function which is executed in a PARALELL way to the main function in the program. In this user customized function any other methods of the class can be called, for example: WaitForButton() without affect the behavior of the main function.

{% highlight cpp%} int BUTTON::WhenButtonWasPressed(callbackType callbackFunction) { if (this->mode != INPUT) { perror("'waitForButton' method only works on INPUT mode"); return -1; } string message = "'WhenButtonWasPressed' method has been activated!!!"; cout << RainbowText(message, "Orange") << endl; whenButtonWasPressedThread = thread(callbackFunction); return 1; } {% endhighlight %}

The second method WaitForButton(int) is an overload of the first. The difference consists that this method receives only psyncone of the next types of transition on the pin:

  • RISING
  • FALLING
  • BOTH

{% highlight cpp%} int BUTTON::WaitForButton(int edge = RISING) { if (this->mode != INPUT) { perror("'waitForButton' method only works on INPUT mode"); return -1; } string message; switch (edge) { case RISING: WriteFile(path, "edge", "rising"); while (this->DigitalRead() != LOW); while (this->DigitalRead() != HIGH); message = "A rising edge was detected!"; cout << RainbowText(message, "Yellow") << endl; break; case FALLING: WriteFile(path, "edge", "falling"); while (this->DigitalRead() != HIGH); while (this->DigitalRead() != LOW); message = "A falling edge was detected!"; cout << RainbowText(message, "Yellow") << endl; break; case BOTH: WriteFile(path, "edge", "both"); int previousValueOnPin = this->DigitalRead(); while (this->DigitalRead() == previousValueOnPin); message = "A rising / falling edge was detected!"; cout << RainbowText(message, "Yellow") << endl; break; } return 1; } {% endhighlight %}

The button can be declared as a BUTTON object specifying the pin attached to it:

{% highlight cpp%} BUTTON redButtonPin(P8_08); {% endhighlight %}

In the main program, only a single line makes to wait for the press of a button, before continue with the next line of code:

{% highlight cpp%} redButtonPin.WaitForButton(); {% endhighlight %}

Complete code
